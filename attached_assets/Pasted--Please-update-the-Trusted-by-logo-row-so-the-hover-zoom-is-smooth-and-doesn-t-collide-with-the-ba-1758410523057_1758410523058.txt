“Please update the ‘Trusted by’ logo row so the hover zoom is smooth and doesn’t collide with the background logos. Wrap each logo image in a fixed-size container and only scale the image inside. The container should have overflow-hidden so the image never spills out. Use transform-gpu, will-change: transform, and a 300–500ms ease-out transition for smoothness. On hover, raise the z-index of the card so it visually sits above neighbors (without reflow) and add a subtle shadow. Keep the decorative background logos, but reduce their opacity and ensure they sit on a separate absolutely-positioned layer. Also fade that background layer slightly (e.g., opacity 40% → 30%) when any logo is hovered to reduce visual conflict.”

<script>
// reduce background prominence when any logo is hovered
const section = document.currentScript.closest('section');
const bg = section.querySelector('div.absolute');
const grid = section.querySelector('#trusted-logos');

grid.addEventListener('mouseover', () => { bg.style.opacity = '0.30'; });
grid.addEventListener('mouseout',  () => { bg.style.opacity = '0.40'; });

// ensure hovered card stacks above neighbors without layout shift
grid.addEventListener('mouseover', e => {
  const card = e.target.closest('.group');
  if (card) card.style.zIndex = 10;
});
grid.addEventListener('mouseout', e => {
  const card = e.target.closest('.group');
  if (card) card.style.zIndex = '';
});
</script>

why this works

the image scales, not the grid cell → no layout shift.

the card has overflow-hidden → no spill/overlap with neighbors.

transform-gpu + will-change + ease-out 500ms → smooth zoom.

temporary higher z-index on hover gives a clean “pop” without affecting layout.

fading the background logo layer during hover reduces the visual clash you were seeing.
